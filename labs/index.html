CameraControl

Путь: scripts/CameraControl.gd

Наследование: Node3D < Node < Object

Описание: Скрипт управления камерой от третьего лица (TPS). Поддерживает управление мышью, геймпадом и автоматический поворот за игроком.

Экспортируемые свойства

PlayerCharacterMesh @export
Тип: NodePath

Путь к узлу меша игрока для ориентации камеры

Группа: "Настройки ограничений" @export_group

cam_v_max @export
Тип: int

Максимальный угол наклона камеры вверх (рекомендуется 75)

cam_v_min @export
Тип: int

Минимальный угол наклона камеры вниз (рекомендуется -55)

Группа: "Чувствительность" @export_group

joystick_sensitivity @export
Тип: int

Множитель чувствительности для правого стика геймпада

Группа: "Плавность" @export_group

Внутренние свойства

Методы

_ready()

Описание: Инициализация камеры при загрузке сцены.

Выполняемые действия:

  • Захват курсора мыши внутри окна игры
  • Исключение родительского узла (игрока) из коллизий камеры (RayCast)
_input(event)

Параметры:

  • event - событие ввода

Описание: Обработка движения мыши для вращения камеры.

Логика:

  • Запускает таймер задержки для отключения авто-поворота при ручном управлении
  • Обновляет углы поворота камеры на основе движения мыши с учетом чувствительности
_joystick_input()

Описание: Обрабатывает ввод с джойстика/геймпада для вращения камеры.

Используемые действия:

  • lookleft / lookright
  • lookup / lookdown

Логика:

  • Запускает таймер задержки при активном вводе
  • Вычисляет вектор направления на основе силы нажатия стиков
  • Обновляет углы поворота камеры с учетом чувствительности джойстика
_physics_process(delta)

Параметры:

  • delta (float) - время с последнего физического кадра

Описание: Основной цикл обработки поведения камеры.

Выполняемые действия:

  • Опрос ввода с геймпада
  • Ограничение вертикального вращения (предотвращение "мертвой петли")
  • Расчет скорости автоматического следования камеры за мешем персонажа
  • При отсутствии ручного ввода - плавное следование камеры за игроком
  • При ручном управлении - применение сглаживания к углам поворота
  • Применение вертикального вращения к узлу наклона

Логика работы

Режимы управления:

  1. Ручное управление (мышь/геймпад): Камера поворачивается в соответствии с вводом пользователя. Активируется таймер задержки для временного отключения авто-поворота.
  2. Автоматическое следование: Когда таймер задержки истекает и нет активного ввода, камера плавно следует за направлением меша персонажа. Скорость следования зависит от скорости движения персонажа.

Система сглаживания:

Все повороты камеры применяются через функцию lerp для плавности движения. Скорость сглаживания регулируется параметрами h_acceleration и v_acceleration.

Ограничения:

Вертикальное вращение камеры ограничено диапазоном от cam_v_min до cam_v_max для предотвращения неестественных поворотов.

Требования к сцене

  • Узел $h - для горизонтального вращения
  • Узел $h/v - для вертикального наклона
  • Узел $h/v/Camera3D - сама камера
  • Узел $control_stay_delay - таймер для задержки переключения режимов
  • Родительский узел должен иметь свойство horizontal_velocity
